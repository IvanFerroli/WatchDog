# ADR Surface

## Objetivo
Registrar decisoes arquiteturais aceitas com contexto, alternativa e impacto rastreavel.

## Quando usar
- decisao de arquitetura com impacto transversal;
- politica estrutural que precisa de historico de revisao;
- excecao arquitetural relevante.

## Convencao minima
- ID: `ADR-###`
- Arquivo por ADR: `ADR-###-<slug>.md`
- Base inicial: `docs/adr/_template.md`

## Campos obrigatorios
- `status`
- `owner`
- `last-updated`
- `source-of-truth`

## Fora de escopo
- implementar codigo;
- escrever spec completa;
- registrar brainstorming sem decisao.
